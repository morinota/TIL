  - ChatGPTの回答
    - 「MLパイプラインがデータ収集、特徴量エンジニアリング、モデルのトレーニング、モデルの評価、モデルのデプロイ、モデルの監視、推論、メンテナンスを行う」
    - ↑は、素晴らしくて魔法のような単一のパイプライン! 
  - 10人の異なるデータサイエンティストに同じ質問をしたら、10通りの異なる回答が返ってくる。
    - なので著者の意見では、少なくとも現在の用語の使われ方では、MLパイプラインという用語はAIシステムの構築についてコミュニケーションを撮る際に有害にもなり得る。
    - よって、一旦はMLパイプラインの定義を保留！
  - "Pipeline"とは??
    - >明確に定義された入力と出力（つまり、明確に定義されたインターフェースを持つ）を持ち、スケジュールに従ってまたは継続的に実行されるコンピュータプログラム。
    - MLパイプラインはAIシステムで使用される何らかのML成果物を出力する任意のパイプライン! 
      - ML成果物(ML artifact) = モデル, 特徴量, 学習データ, ベクトルインデックス, モデルデプロイメント, 予測/コンテキストログなど。
    - **よし! 各MLパイプラインを、それが作成(create) or 変更(modify) するML成果物(ML artifact)に基づいて名付けよう!**
      - ML成果物をcreateするパイプラインの例:
        - 特徴量を出力するfeature pipeline
        - 埋め込みを出力する vecgor embedding pipeline
        - 学習されたモデルを出力する training pipeline
        - 予測を出力する inference pipeline
      - ML成果物をmodifyするパイプラインの例:
        - モデルを未検証から検証済みに変更する model validation pipeline
        - モデルを本番環境にデプロイする model deployment pipeline
