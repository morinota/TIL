# contexual bandit × ランク学習の手法「BanditRank」の論文を読んだ!

- 深層学習ベースの機械学習モデルを、**contexual bandt的に定式化した上でランク学習する方法「BanditRank」**を提案した論文。
  - 要するに、**任意のニューラルネットモデルを強化学習ベースでランク学習(Learning For Rank)できる手法を提案してるよ**、って話:thinking:
- この論文のメインのドメインは情報検索(IR)。

## 背景・本論文のモチベーションの話

- ランク学習(Learning To Rank)は、情報検索(IR)や推薦システムで重要な役割を果たす技術分野。
  - ランク学習で使用される学習アプローチは、全て以下の3カテゴリに分類できる。
    - ポイントワイズ(pointwise)
    - ペアワイズ(pairwise)
    - リストワイズ(listwise)
- ランク学習の目的関数はいろいろあるが、**IR指標(=MAPやnDCGなど?)の直接最適化は、ランキング学習の分野における長年の課題**...!
  - 直接最適化のための学習アルゴリズムは、以下の3カテゴリに分類できる:
    - 1番目: IR指標の上限や近似を用いて代理目的関数を最適化する方法。
    - 2番目: IR指標の真の勾配を近似するような推定値を使用する方法 (LambdaRankなど)。
    - 3番目: 強化学習(RL)を用いて、近似なしに報酬の形でIR指標を直接最適化する方法。(MDPRankなど)
  - **しかし、LambdaRankのような一部のアルゴリズムを除いて、すべてのカテゴリのほとんどのアルゴリズムは、パラメータが少ないモデルにのみ適しており（Donmez et al., 2009）、非常に効果的な深層ニューラルネットワークを使用することが難しくなる**。
  - 実際、深層ニューラルネットワークを使用したランキングモデルは、通常、クロスエントロピー損失などの間接的な目的関数で訓練されがち。
    - CTR予測のような間接的な目的関数は、IRの評価指標に関する情報は持ってない。
    - **MAP、MRR、およびnDCGなどのほとんどのIR評価指標は微分可能ではないため、ニューラルネットワークの訓練のための目的関数としてそのまま使用できない**。
