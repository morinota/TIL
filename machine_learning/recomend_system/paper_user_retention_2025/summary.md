# Save, Revisit, Retain: 大規模レコメンダーシステムにおけるユーザーretention向上フレームワーク

## 元記事情報

- **タイトル**: Save, Revisit, Retain: A Scalable Framework for Enhancing User Retention in Large-Scale Recommender Systems
- **著者**: Weijie Jiang, Armando Ordorica, Jaewon Yang, Olafur Gudmundsson, Yucheng Tu, Huizhong Duan (Pinterest Inc)
- **出典**: [arXiv:2511.18013](https://www.arxiv.org/pdf/2511.18013)
- **実装環境**: Pinterest Related Pins (5億人以上のユーザー)

## 概要

Pinterestが開発した、レコメンダーシステムにおける長期的なユーザーretentionを最適化するための新しいフレームワーク。従来の即時的なエンゲージメント指標(クリック、保存)ではなく、「再訪行動(revisitation)」に注目し、保存したコンテンツへの再訪をretentionの代理指標として活用する点が特徴。

本フレームワークは、マルチタスク学習を用いた軽量で解釈可能な設計であり、追加の計算コストなしにアクティブユーザーを0.1%増加させることに成功した。

## Pinterest用語集

本論文を理解するために必要なPinterestの基本用語を整理します。

### 基本概念

- **ピン(Pin)**: Pinterest上の画像・動画などのコンテンツの単位。レシピ、ファッション、インテリアなど様々なトピックの画像が「ピン」として投稿される
- **ボード(Board)**: ピンを保存・整理するためのコレクション。ユーザーが自分の興味に応じてテーマ別にピンをまとめる場所
- **リピン(Repin/Save)**: 他のユーザーのピンや推薦されたピンを自分のボードに保存する行動。本論文の中心的な行動の1つ
- **プロフィール**: ユーザーが自分のボードや保存したピンを管理・閲覧する個人ページ

### 表示と操作

- **グリッド表示**: ピンが格子状(グリッド状)に並んで表示されるUI。Pinterestの特徴的な表示形式
- **グリッドクリック(Grid-click)**: グリッド表示されているピンの1つをタップして拡大・詳細表示すること。ピンに対する能動的な興味を示す行動
- **インプレッション(Impression)**: コンテンツが画面上に表示されること。グリッドをスクロールして見て回るだけで、タップはしない受動的な閲覧

### サーフェス(Surface)

Pinterestにおける「サーフェス」とは、アプリ内の異なる画面・機能領域のこと。

- **Related Pins surface**: あるピンをグリッドクリックしたときに表示される画面
  - 上部: クリックしたピンの詳細(タイトル、説明、保存ボタン)
  - 下部: "More like this"(これに似たもの)として関連ピンがグリッド表示される
  - Pinterest全体のトラフィックの約50%を占める重要な推薦サーフェス
- **Homefeed**: ユーザーがアプリを開いたときに最初に表示されるフィード
- **Search**: 検索結果を表示するサーフェス
- **Profile**: ユーザー自身の保存したピンやボードを閲覧するサーフェス

### 再訪行動(本論文の中心概念)

- **再訪(Revisitation)**: ユーザーが一度保存したピンを、後日自分のプロフィールで再度閲覧すること
  - **インプレッションベースの再訪**: 保存したピンのグリッドを眺める(スクロールやボード間移動)だけで、タップはしない
  - **グリッドクリックベースの再訪**: 保存したピンを実際にタップして拡大・詳細表示する。より能動的で深いエンゲージメント

## 背景と課題

### なぜretentionが重要なのか

ユーザーretentionは、Pinterestのようなオンラインプラットフォームにとって重要なビジネス目標。ユーザーの忠誠心を強化し、繰り返しのエンゲージメントを通じて成長を促進する。

#### retentionの代理指標としての「再訪(revisitation)」

**再訪とは**: ユーザーが以前に保存したコンテンツを、後日再度閲覧する行動

**なぜ再訪が重要な指標なのか**:
- ユーザーが満足したコンテンツを保存している
- 保存したコンテンツに価値を感じて戻ってくる
- 単なる偶然の訪問ではなく、意図的な再利用

→ 再訪行動が多いユーザーほど、長期的にプラットフォームを利用し続ける傾向がある

### 再訪行動のモデル化における3つの課題

#### 課題1: 帰属(attribution)の問題 - 何が再訪を引き起こしたのか?

**問題**: ユーザーが再訪したとき、それが「どのタイミングでの何の行動」によって引き起こされたのかを特定するのが困難

**具体例**:

```
月曜: ユーザーAがレシピのピンを保存
火曜: ユーザーAがファッションのピンを閲覧(保存なし)
水曜: ユーザーAがプロフィールでレシピのピンを再訪
```

この場合:
- ✅ 月曜の保存が再訪を引き起こした → 正しい帰属
- ❌ 火曜のファッション閲覧が再訪を引き起こした → 誤った帰属

**混乱要因が多数存在**:
- プッシュ通知が届いた
- メールリマインダーを見た
- UIのデザインが変わった
- 友達に勧められた
- まったく別の目的でアプリを開いたら、たまたま以前のピンが目に入った

→ これらの要因を考慮しないと、間違った因果関係を学習してしまう

#### 課題2: スケールの問題 - データ量が膨大すぎる

**問題**: 再訪は数日〜数週間後に発生するため、長期間のデータを保持・結合する必要がある

**具体的なスケール感**:

Pinterestの場合:
- ユーザー数: 5億人以上
- 1日の保存数: 数億件
- 追跡期間: 1週間〜1ヶ月
- 結合対象: 保存イベント × 再訪イベント

→ 単純計算で数兆件のイベントを処理・結合する必要がある

**技術的な課題**:
- ストレージ: 数週間分のユーザー行動ログを保持
- 計算: ユーザーID × ピンID × 時間範囲でのクロス結合
- レイテンシ: リアルタイム推薦に影響を与えないよう高速処理が必要

→ 既存のバッチ処理やリアルタイム処理の仕組みでは限界がある

#### 課題3: 既存手法の限界 - なぜ従来のアプローチでは解決できないのか

**アプローチ1: 即時エンゲージメント指標に最適化**

多くの既存のレコメンダーシステムは以下を最適化:
- クリック率(CTR)
- 保存率
- セッション時間

問題: これらは短期的な指標であり、長期的なretentionとの相関が必ずしも高くない

例:
- クリックベイト的なコンテンツ → 高CTRだが、ユーザー満足度は低い
- 即座に消費するコンテンツ → 保存されるが、再訪されない

**アプローチ2: 強化学習(RL)で長期的な報酬を最適化**

RLは理論的には長期的な報酬を最適化できるが、現実の大規模システムでは困難:

1. **時間スケールの問題**:
   - RL得意領域: セッション内(数分〜数時間)の行動最適化
   - 再訪問題: 数日〜数週間後のイベント

   ```
   セッション内 (RL得意)
   ├─ 推薦 → クリック → 次の推薦 → クリック
   └─ フィードバックが即座に得られる

   再訪問題 (RL苦手)
   ├─ 推薦 → 保存 → (3日間の空白) → 再訪
   └─ フィードバックが遅延、状態が変化しすぎ
   ```

2. **状態空間の爆発**:
   - 数日間にわたる数千件の推薦アイテム
   - ユーザーの状態(興味、文脈)が日々変化
   - 状態空間が膨大になり、学習が困難

3. **報酬の希薄性(sparsity)**:
   - 保存されるピンは全体の数%
   - 再訪されるピンはさらにその一部
   - ほとんどのアクションに対して報酬がゼロ → 学習が進まない

4. **スケーラビリティの欠如**:
   - Pinterestスケール(5億人)でRLを運用した事例がない
   - 計算コストとレイテンシが実用レベルに達していない

→ 既存のRL手法では、大規模プラットフォームにおける再訪最適化は実現できていない

## 提案フレームワークの設計

### 1. 再訪行動の分析

Pinterestにおける再訪行動の大規模分析から、以下のパターンを発見:

**再訪の種類**

- **インプレッションベースの再訪(Impression-based Revisitation)**: 保存されたコンテンツを閲覧(スクロールやボード間の移動)するが、深いインタラクションはしない
- **グリッドクリックベースの再訪(Grid-click-based Revisitation)**: 保存されたコンテンツをタップして詳細情報にアクセス

**発見されたパターン**

1. **再訪するユーザーの割合は時間とともに減少する**

   ピンを保存したユーザーのうち、何%が後日そのピンを再訪するかを追跡:

   - **インプレッションベースの再訪**:
     - 保存当日(0日目): 14.6%のユーザーが再訪
     - 1日後: 19.5%(ピーク)
     - 9日後: 8.7%まで減少

   - **グリッドクリックベースの再訪**:
     - 保存当日(0日目): 6.6%のユーザーが再訪
     - 9日後: 1.6%まで急激に減少

2. **再訪されるピンの割合は最初の3日間で劇的に減少する**

   保存された全ピンのうち、何%が再訪グリッドクリックを受けるかを追跡:

   - 保存当日(0日目): 4.7%のピンが再訪される
   - 1日後: 約2.4%(半分に減少)
   - 3日後: 約1.2%(さらに半分に減少)
   - 5日後: 0.5%
   - 9日後: 0.3%

   → **保存直後の3日間が最も重要な期間**

3. **早く再訪するユーザーほど、その後もアクティブに利用する**

   ピン保存後の再訪タイミングと、その後1か月間のアクティブ日数の関係を分析:

   - 早期(0〜2日以内)に再訪したユーザー: 次の1か月間のアクティブ日数が多い
   - 再訪しなかったユーザー: 次の1か月間のアクティブ日数が少ない

   → **早期の再訪は、長期的なユーザーretentionの強い予測因子**

4. **グリッドクリックによる再訪の方が、インプレッションよりも深いエンゲージメントを示す**

   2種類の再訪行動と、その後1か月間のアクティビティ変化の相関を比較:

   - **グリッドクリックベースの再訪**: 1か月間のアクティビティ変化との相関が高い
   - **インプレッションベースの再訪**: 1か月間のアクティビティ変化との相関が低い

   → **タップして詳細を見る(グリッドクリック)という能動的な行動の方が、ユーザーの真の興味を反映している**

### 2. 代理帰属プロセスの設計:「どの保存が再訪を引き起こしたか」を定義する

#### 課題

再訪が発生したとき、それがどのタイミングでの保存によるものかを特定する必要がある

#### 解決策

保存と再訪の時間的関係で帰属を定義:

- 同日再訪: ユーザーがRelated Pinsでピンを保存 → 同日中にプロフィールで再訪 → その保存に帰属
- 複数日再訪: ユーザーがRelated Pinsでピンを保存 → 数日後にプロフィールで再訪 → その保存に帰属

重要なポイント: 他の日に見た無関係なピンは、保存したピンの再訪に帰属させない

### 3. 再訪ラベル設計:機械学習モデルが学習するための正解データを作る

#### 設計目標

- カバレッジ(ラベル密度): できるだけ多くのデータポイントで再訪ラベルを付与
- 精度(意図): ノイズの少ない、真の再訪意図を反映したラベル

#### 採用された3種類の再訪ラベル

1. 同日再訪インプレッション(1dRevImpre)
   - 定義: ピンを保存 → 同日中にプロフィールで閲覧(スクロールして見る)
   - カバレッジ: リピン(保存)の25.8%
   - 特徴: 高カバレッジだが、偶発的なスクロールも含む可能性

2. 同日再訪グリッドクリック(1dRevGrid)
   - 定義: ピンを保存 → 同日中にプロフィールでタップして詳細表示
   - カバレッジ: リピンの4.7%
   - 特徴: 能動的な行動なので意図が明確

3. 7日再訪グリッドクリック(7dRevGrid)
   - 定義: ピンを保存 → 0〜6日後にプロフィールでタップして詳細表示
   - カバレッジ: リピンの9%
   - 特徴: 長期的な興味を反映、累積効果が高い

統合: 3種類のラベルをOR条件で統合 → いずれかの再訪があれば正例(ポジティブラベル)

### 4. 再訪特徴の設計:ピンの「再訪されやすさ」を数値化する

#### Pin Perf (Pin Performance)フィーチャーとは

あるピンが過去にどれだけユーザーから再訪されたかを集計した人気指標。

#### 設計した再訪Pin Perfフィーチャー(5種類)

| フィーチャー名 | 定義 | カバレッジ | 用途 |
|---|---|---|---|
| 同日再訪インプレッションPin Perf | Related Pinsで保存され、同日中に再訪閲覧された回数 | 60% | ラベルと対応する短期的な再訪人気 |
| 同日再訪グリッドクリックPin Perf | Related Pinsで保存され、同日中に再訪クリックされた回数 | 49% | ラベルと対応する短期的な能動的再訪 |
| 7日再訪グリッドクリックPin Perf | Related Pinsで保存され、0〜6日後に再訪クリックされた回数 | 51% | ラベルと対応する長期的な再訪人気 |
| 全体再訪インプレッションPin Perf | すべてのサーフェスでの再訪閲覧の一般的人気 | 80% | 汎用的な再訪されやすさ |
| 全体再訪グリッドクリックPin Perf | すべてのサーフェスでの再訪クリックの一般的人気 | 70% | 汎用的な能動的再訪されやすさ |

#### 集計期間

過去7日間(毎日更新)、30日間(3日ごと更新)、90日間(毎週更新)

#### 各フィーチャーの内訳

- 「アクションの数」: 再訪された総回数(ボリューム)
- 「ユニークユーザーの数」: 何人のユーザーが再訪したか(人気の広がり)

→ 同じピンを1人のユーザーが何度も再訪している場合と、多くのユーザーが再訪している場合を区別

### 5. マルチタスク学習推薦システムへの統合:既存モデルに再訪予測を追加

#### 既存のマルチタスク推薦モデル

従来から以下の行動を同時に予測:
- グリッドクリック
- リピン(保存)
- クリック(外部サイト訪問)
- 長クリック(外部サイトに35秒以上滞在)

#### モデルアーキテクチャ

```
入力層
  ├─ クエリピン特徴(ユーザーがクリックしたピンの情報)
  ├─ 候補ピン特徴(推薦候補となるピンの情報)
  ├─ ユーザー特徴(ユーザーの属性や興味)
  └─ ユーザー行動履歴(Transformerでエンコード)
     ↓
要約層
  └─ 特徴交差 + DCNv2(深い特徴の相互作用を学習)
     ↓
MMoE(Multi-gate Mixture of Experts)
  └─ 各タスク(グリッドクリック、リピン、再訪など)ごとに専用の予測ヘッド
```

#### 追加された再訪予測タスク

学習時の損失関数:
```
loss2 = loss1(既存タスク) + w_{RP&RV} × 再訪予測の損失
```

- `w_{RP&RV}`: 損失重み(loss weight) - 学習時に各タスクの損失をどれだけ重視するかを調整
- 再訪タスクの損失をどの程度学習に反映させるかを決定

推論時のランキングスコア:
```
score = Σ(既存タスクのスコア) + u_{RP&RV} × p(保存&再訪|クエリピン)
```

- `u_{RP&RV}`: ユーティリティ重み(utility weight) - 推論時に各タスクの予測をどれだけ重視するかを調整
- 再訪予測をランキングスコアにどの程度反映させるかを決定

#### 最適なハイパーパラメータ

実験の結果、以下の設定が最適:
- ユーティリティ重み: `u_{RP&RV} = 1.27 × u_{Repin}`
- 保存(Repin)の1.27倍の重要度で再訪を評価することで、最も良い結果が得られた

補足:
- 損失重み(w)とユーティリティ重み(u)は別々に調整可能
- w: モデルがどう学習するか(学習時のバランス)
- u: モデルがどう予測するか(推論時のバランス)

→ 追加の推論コストなし: 既存のマルチタスクモデルに1つのヘッドを追加するだけ

### 6. クロスサーフェス再訪データパイプライン:異なる画面のデータを結合する

#### 課題

- ピンの保存はRelated Pinsサーフェスで発生
- 再訪はProfileサーフェスで発生
- この2つを結びつける必要がある

#### データパイプラインの5ステップ

```
Step 1: 再訪Pin Perf特徴を生成
  └─ 過去の再訪履歴から各ピンの人気指標を計算
     ↓
Step 2: Related Pinsサーフェスのログからラベル抽出
  └─ ユーザーID、ピンID、リピン(保存)行動を記録
     ↓
Step 3: クロスサーフェス・クロスセッション結合 ★核心
  └─ 結合条件:
      - ユーザーIDが一致
      - ピンIDが一致
      - time_save < time_revisit < time_save + 7日
  └─ Related Pinsでの保存 ← 結合 → Profileでの再訪
     ↓
Step 4: 再訪ラベルを既存のリピンラベルと統合
  └─ 「保存されたピン」に「再訪されたかどうか」のフラグを追加
     ↓
Step 5: 特徴とラベルを結合して最終的な学習データを生成
  └─ APIリクエストID、候補ピンIDで結合
```

#### スケーラビリティ

- Step 1, 2, 3は並列実行可能
- Step 4, 5は順次実行
- 7日間のウィンドウで集約することで計算量を抑制
- 他のプラットフォーム(保存→再訪の概念があるサービス)にも適用可能

## 実験結果

### オフライン実験:再訪モデルの効果を事前検証

#### データセット

- トレーニング: 27日間のデータ(約66億サンプル)
- 評価: 3日間のデータ(約7億サンプル)
- 1エポックでトレーニング

#### ランキング精度の改善

従来モデル(再訪なし)と提案モデル(再訪あり)の比較:

| メトリック | Repin | Revisit |
|-----------|--------|---------|
| NDCG@3 Lift (%) | 0.13% | 40.15% |
| MAP@3 Lift (%) | 0.06% | 43.44% |
| Recall@3 Lift (%) | 0.27% | 35.4% |
| Pairwise Accuracy Lift (%) | 0.10% | 21.62% |
| Hits@3 Lift (%) | 0.59% | 0.65% |

#### 結果の解釈

1. 再訪予測の精度が大幅に向上(20%-40%のリフト)
   - 再訪されやすいピンを上位にランキングできるようになった

2. 保存(Repin)の予測精度も副次的に向上(0.05%-0.27%のリフト)
   - すべての再訪されたピンは保存されたピンなので、保存予測も改善

3. Hits@3が最も重要な指標
   - Hits@3: 上位3件の推薦のうち、少なくとも1件がユーザーに保存・再訪されたか
   - Pinterestの経験上、Hits@3がオンラインA/B実験の結果と最も相関が高い

### オンラインA/B実験:実際のユーザーへの効果を検証

#### 実験設計

- 期間: 2025年4月29日～6月26日(2ヶ月間)
- ユーザー数: 各グループ約1,200万人(合計2,400万人)
- 対象: Pinterest Related Pinsサーフェス
- 比較: 対照群(従来モデル) vs 処置群(再訪モデル)

#### 1. 再訪行動への直接的な影響

再訪モデルは、狙い通り再訪行動を増加させた:

- 再訪グリッドクリック全体: 0.95% - 1.42%の増加
- 7日再訪グリッドクリック: 同日再訪よりも大きな増加
  → 長期的な再訪行動を推進する強い累積効果
- 再訪比率の向上:
  - ボリューム(絶対数): 保存からの再訪が増えた
  - 傾向(率): 保存されたピンのうち再訪される割合が高まった

#### 2. エンゲージメントへの波及効果

再訪行動の増加により、他のエンゲージメント指標も改善:

| 指標 | 改善率 |
|---|---|
| リピン(保存)ボリューム | +0.94% |
| リピン(保存)傾向 | +0.64% |
| 5分以上のセッション | +0.41% |
| Web/APIリクエスト | +0.35% |

#### 3. ユーザー滞在時間の増加

ユーザーがPinterest上でより長く滞在するようになった:

| サーフェス | 滞在時間の増加 |
|---|---|
| Related Pinsサーフェス | +0.53% |
| プロフィール(再訪が発生する場所) | +0.68% |
| アプリ全体 | +0.39% |

#### 4. 最終的なビジネスインパクト:アクティブユーザーの増加

- アクティブユーザーボリューム: +0.1%
- アクティブユーザー傾向: +0.08%

5億人規模のプラットフォームで0.1%の増加は、約50万人のアクティブユーザー増加に相当。

#### 重要な洞察

モデルはRelated Pinsサーフェスの推薦にのみ直接影響を与えたが、プロフィールやアプリ全体のretentionメトリクスも改善した。

これは、Related Pinsでの良質な保存体験が、後日のプロフィールでの再訪を促進し、最終的にプラットフォーム全体での活動を活性化させたことを示す。

## 洞察と解釈可能性

本フレームワークは、なぜ効果があるのかを説明できる解釈可能性を持つ。

### トピック別のrepin率 vs 再訪率:コンテンツ特性による違い

Pinterestの16の主要トピックを分析した結果、トピックによって保存行動と再訪行動に明確な傾向の違いがあることが判明。

#### パターン1: たくさん保存されるが、あまり再訪されない

**トピック**: DIY and Crafts、Parenting、Health

**特徴**: 実生活で実践するために保存するが、一度実行したら再訪の必要性が低い

**例**:
- DIYのレシピを保存 → 作ったら完了
- 育児のヒントを保存 → 試したら終わり
- 健康管理の方法を保存 → 実践したら見返さない

#### パターン2: あまり保存されないが、よく再訪される

**トピック**: Beauty、Architecture、Travel、Event Planning、Art、Electronics

**特徴**: 美的な楽しみや計画のために何度も見返す

**例**:
- 美容やファッションの写真 → 何度も見てインスピレーションを得る
- 旅行先の写真 → 計画を練るために繰り返し閲覧
- イベント計画 → 準備期間中に何度も確認

### 長期的 vs 短期的な再訪:トピックによる時間軸の違い

保存から再訪までの期間を分析した結果、トピックによって「すぐ見返す」か「後でじっくり見返す」かが異なることが判明。

指標: `revisit_{3-6day}` = (3〜6日後の再訪数) / (0〜6日後の再訪数の合計)

#### 長期的な再訪が多いトピック(比率が高い = 保存後3〜6日後にじっくり見返す)

| トピック | 比率 | 理由 |
|---|---|---|
| Event Planning | 0.1881 | イベントは数日〜数週間後なので計画期間が長い |
| Health | 0.1836 | 健康管理は継続的に取り組むため |
| Home Decor | 0.1780 | 模様替えの計画に時間がかかる |
| DIY and Crafts | 0.1733 | 材料を揃えてから作業するので準備期間が必要 |

#### 短期的な再訪が多いトピック(比率が低い = 保存後すぐに見返す)

| トピック | 比率 | 理由 |
|---|---|---|
| Finance | 0.1056 | 投資判断など緊急性が高い |
| Electronics | 0.1079 | 購入決定が早い |
| Vehicles | 0.1334 | 車の購入は比較検討が短期集中 |
| Architecture | 0.1495 | 建築デザインは即座のインスピレーション |
| Art | 0.1572 | アート作品は美的満足を即座に得る |

### モデルの解釈可能性:なぜ特定のトピックで効果が高いのか

再訪モデルは、過去に再訪されやすかったピンを優先的に推薦する。

#### 効果が高かったトピック

**Beauty、Architecture、Entertainment**

- モデルが予測する再訪確率 `P(保存&再訪|クエリピン)` が高い
- これらのトピックのピンを推薦した結果、保存(repin)が大きく増加(ポジティブなリフト)
- 理由: もともと再訪されやすいコンテンツなので、ユーザーが満足して保存する

#### 効果が低かった(逆効果だった)トピック

**Finance、Health、Quotes**

- モデルが予測する再訪確率 `P(保存&再訪|クエリピン)` が低い
- これらのトピックのピンを推薦した結果、保存(repin)が減少(ネガティブなリフト)
- 理由: 再訪されにくいコンテンツを優先したため、ユーザーの即座のニーズに合わず保存されなかった

**示唆**: モデルは「再訪されやすさ」を学習しており、トピックごとの特性を正しく捉えている

## 技術的な学び

### 1. 代理指標の重要性:直接測定できない目標を間接的に最適化

#### 課題

長期的なユーザーretention(何ヶ月も先のアクティビティ)を直接最適化するのは困難:
- 因果関係が複雑(何が原因で再訪したか不明)
- フィードバックが遅い(数週間〜数ヶ月後)
- データが疎(ほとんどのユーザーは再訪しない)

#### 解決策

適切な代理指標(surrogate)を選ぶ:
- **代理指標**: 再訪行動(保存後0〜6日以内の再訪)
- **真の目標**: 長期的なユーザーretention(1ヶ月後のアクティビティ)

#### 効果

本研究では、再訪という適切な代理指標を選ぶことで:
- 測定しやすい(保存後すぐにわかる)
- 因果関係が明確(保存→再訪の関係)
- 長期的なretentionとの相関が高い

→ レコメンダーシステムにおいて、適切に選択されたユーザー行動は長期的なエンゲージメントを予測し最適化するための効果的な代理報酬として機能する

### 2. 軽量設計の重要性:追加コストゼロで効果を上げる

#### 一般的なアプローチの問題

新しいタスクを追加する際、別モデルを追加すると:
- **推論コスト**: 追加のモデル推論が必要 → レイテンシ増加
- **運用コスト**: 複数モデルの管理が複雑化
- **スケーラビリティ**: 5億人規模では致命的

例: Zhang et al. (2021)は再訪予測用に別モデルを展開したが、コストとレイテンシが増加

#### 本研究のアプローチ

既存のマルチタスクモデルに1つのヘッドを追加するだけ:
- **追加の推論コストなし**: 同じ特徴表現を共有
- **運用が簡単**: 1つのモデルで完結
- **スケーラブル**: 5億人でも問題なし

→ 大規模プロダクションシステムでは、軽量な設計が実用上非常に重要

### 3. スケーラビリティの設計:他プラットフォームへの適用可能性

#### 汎用的なデータパイプライン設計

本研究のクロスサーフェス・クロスセッション結合の仕組みは、以下の条件があれば適用可能:

1. ユーザーがコンテンツを「保存」できる
2. 保存したコンテンツを後で「再訪」できる
3. ユーザーID、コンテンツID、タイムスタンプを記録している

#### 適用可能な他プラットフォームの例

- YouTube: 「後で見る」リスト → 実際に視聴
- Twitter(X): ブックマーク → 後で閲覧
- LinkedIn: 保存した投稿 → 再閲覧
- Spotify: プレイリスト保存 → 再生
- Instagram: 保存した投稿 → 後で閲覧

→ 本フレームワークは、保存→再訪の概念がある任意のプラットフォームに適用できる

### 4. 時間ウィンドウの選択:データ分析に基づく意思決定

#### データ駆動の設計プロセス

1. **大規模データ分析**: 数億件の保存・再訪イベントを分析
2. **パターン発見**:
   - 再訪の大部分(約80%)が最初の3日以内に発生
   - 早期再訪ほど長期的なretentionとの相関が高い
3. **設計への反映**: 0〜6日の7日間ウィンドウを採用

#### 7日間ウィンドウのメリット

- **シグナルの質**: 重要な再訪の大部分をカバー
- **計算効率**: 無期限ではなく7日間に限定することで計算量を抑制
- **実用性**: DWH(データウェアハウス)で7日間のデータ保持は現実的

→ 闇雲に長い期間を設定するのではなく、データ分析に基づいて最適な時間ウィンドウを選択することが重要

## 今後の課題と展望

### 現在のボトルネック:データの不均衡問題

#### 問題の構造

再訪ラベルの生成には「保存(repin)」が前提条件となる:

```
保存しないユーザー → 再訪ラベルが生成されない
保存するユーザー → 再訪ラベルが生成される可能性がある
```

#### 具体的な問題

1. **ヘビーユーザーへの偏り**

   - 非常にアクティブなユーザー:
     - 1日に数十件のピンを保存
     - 再訪ラベルデータが豊富
     - モデル学習への貢献が大きい

   - あまりアクティブでないユーザー:
     - 週に数件しか保存しない
     - 再訪ラベルデータが限られる
     - モデル学習への貢献が小さい

2. **影響**

   - モデルがアクティブユーザーの行動パターンに最適化される
   - ライトユーザーに対する推薦精度が相対的に低下する可能性
   - 長期的にはユーザー層の多様性が考慮されないリスク

### 今後の改善案:暗黙の再訪シグナルの活用

#### 現在の定義(明示的な再訪)

```
ユーザーがピンを保存 → 後日プロフィールで同じピンを閲覧 = 再訪
```

問題: 保存していないユーザーは再訪ラベルが付かない

#### 提案する拡張(暗黙の再訪)

明示的な保存がなくても、類似の興味を示す行動を「暗黙の再訪」として扱う:

**例1: トピック再訪**
```
セッション1(月曜): ユーザーがTravelトピックのピンを閲覧
セッション2(水曜): 同じユーザーが再びTravelトピックのピンを閲覧
→ トピックに対する継続的な興味 = 暗黙の再訪
```

**例2: セマンティック類似性**
```
セッション1: 「モダンなリビングルーム」のピンを閲覧
セッション2: 「ミニマリストなインテリア」のピンを閲覧
→ 意味的に類似したコンテンツへの興味 = 暗黙の再訪
```

**例3: ビジュアル類似性**
```
セッション1: 青いドレスの画像を閲覧
セッション2: 似た青いドレスの画像を閲覧
→ 視覚的に類似したコンテンツへの興味 = 暗黙の再訪
```

#### 期待される効果

- ライトユーザーからもシグナルを収集できる
- データの不均衡が緩和される
- より多様なユーザー層に対応した推薦が可能

## まとめ

### 本研究の位置づけ

本研究は、レコメンダーシステムにおける長期的なユーザーretentionの最適化という困難な課題に対し、実用的で効果的な解決策を提示した。

### 4つの主要な貢献

#### 1. 大規模な再訪行動の実証分析

- **規模**: 数億人のユーザー、数十億件のイベント
- **新規性**: 大規模プラットフォームにおける再訪パターンの初の包括的分析
- **発見**:
  - 再訪の80%が保存後3日以内に発生
  - グリッドクリック再訪は長期retentionとの相関が高い
  - トピックによって再訪パターンが大きく異なる

#### 2. 実用的なフレームワークの設計と実装

- **軽量性**: 既存のマルチタスクモデルに1つのヘッドを追加するだけ
- **スケーラビリティ**: 5億人規模でも追加コストゼロ
- **汎用性**: 保存→再訪の概念があるプラットフォームに適用可能
- **実装**: Pinterestに本番展開済み

#### 3. 高い解釈可能性

従来のブラックボックスな手法と異なり、本フレームワークは:
- なぜ効果があるのかを説明できる
- トピックごとの効果の違いを理解できる
- ビジネス判断に活用できる洞察を提供

例:
- Beautyトピックで効果が高い理由: 美的コンテンツは繰り返し閲覧される
- Financeトピックで効果が低い理由: 即座の意思決定が必要で再訪が少ない

#### 4. 実証されたビジネスインパクト

オンラインA/B実験(2ヶ月間、2,400万ユーザー)で以下を達成:

| 指標 | 改善率 | ビジネス意義 |
|---|---|---|
| アクティブユーザー | +0.1% | 約50万人の増加(5億人規模) |
| 再訪グリッドクリック | +0.95〜1.42% | 長期的なエンゲージメント向上 |
| プロフィール滞在時間 | +0.68% | ユーザー満足度の向上 |

### 実務への示唆

1. **代理指標の選択が重要**: 直接測定できない長期目標は、適切な代理指標を通じて最適化できる

4. **解釈可能性がビジネス価値**: ブラックボックスではなく、説明可能なモデルがビジネス判断を支援
